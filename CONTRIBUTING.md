# Contribution Guidelines

## Coding Standards
- **Indentation**: Use 4 spaces for indentation (no tabs).
- **Variable Naming**: Use camelCase for variable names.
- **Commenting**: Code should be self-explanatory; add comments for complex logic.
- **Function Length**: Keep functions short and focused on a single task.

## Workflow
1. **Fork the Repository**: Start by forking the repo.
2. **Create a Branch**: Make a new branch for your feature or fix.
3. **Make Your Changes**: Code away and ensure your changes follow the standards.
4. **Run Tests**: Execute the tests to ensure your changes don't break anything.
5. **Open a Pull Request**: Once satisfied, open a pull request with a descriptive message.
